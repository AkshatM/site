## Why Rust?

Rust exists because developers were tired of choosing between:

- 🛡 **Safety**: Avoiding bugs like null pointer dereferences or data races.
- ⚡ **Performance**: Running close to the metal without unnecessary overhead.
- ❤️ **Ergonomics**: Writing expressive code without tearing your hair out.

With Rust, you get **all three**.

## Core Principles

Rust builds its identity around these key ideas:

### 1. Ownership  
A unique system that lets the compiler track **who owns what** in memory — without a garbage collector.

### 2. Borrowing & Lifetimes  
You can "borrow" references instead of copying, and the compiler ensures they never outlive the data they point to.

### 3. Concurrency Without Fear  
Thread safety is **enforced at compile time**.

### Syntax Highlights

Here’s a quick Rust snippet that shows off some features:

```rust
fn main() {
    let name = "Rustacean";
    let numbers = vec![1, 2, 3, 4];

    println!("Hello, {}!", name);

    let doubled: Vec<i32> = numbers.iter().map(|x| x * 2).collect();
    println!("Doubled numbers: {:?}", doubled);
}
```

**Inline code example**: Rust’s `match` expression makes pattern matching concise:

```rust
match number {
    0 => println!("Zero"),
    1..=9 => println!("Single digit"),
    _ => println!("Multiple digits"),
}
```

## When to Use Rust

You might choose Rust for:

- Building CLI tools
- Writing web backends (via [Actix](https://actix.rs/) or [Rocket](https://rocket.rs/))
- Systems programming (kernels, drivers)
- Game engines
- Embedded development

## Comparison Table

| Feature               | Rust                    | C++                   | Go                     |
|-----------------------|-------------------------|-----------------------|------------------------|
| **Memory Safety**     | ✅ Compile-time checks   | ❌ Manual management  | ✅ Garbage collected   |
| **Performance**       | ⚡ Near C/C++ speed      | ⚡ Near C speed       | ⚡ Good, but slower    |
| **Concurrency Model** | Ownership + Borrowing   | Manual sync           | Goroutines + Channels |
| **Learning Curve**    | 🟧 Steep but rewarding   | 🟥 Steep             | 🟩 Easy                |
